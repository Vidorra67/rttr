# Release v0.12.3

## Ziel

Hotfix für Web-Setup und Migrationen auf MySQL/MariaDB-Hostings.

## Ursache

MySQL und MariaDB führen DDL-Anweisungen wie `CREATE TABLE` und `ALTER TABLE` mit implizitem Commit aus. Der bisherige MigrationRunner hat jede Migration in eine PDO-Transaktion gelegt. Nach dem impliziten Commit war keine aktive Transaktion mehr vorhanden, wodurch beim abschließenden Commit der Fehler `There is no active transaction` entstehen konnte.

## Umsetzung

Der MigrationRunner erkennt den PDO-Treiber. Bei MySQL/MariaDB werden Migrationen ohne äußere Transaktion ausgeführt. Die Migrationen bleiben über `CREATE TABLE IF NOT EXISTS`, Spaltenprüfungen und `INSERT IGNORE` wiederholbar.

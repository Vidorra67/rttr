# v0.14.7 – Ranghistorie und Importlogik korrigiert

## Umsetzung

- Die Anmeldehistorie wird beim Import jetzt nicht mehr nur nach dem Zieljahr gelesen.
- Pro Person wird der letzte bekannte Rang bis zum Quelljahr verwendet.
- Dadurch wird eine Person nicht mehr fälschlich auf einen alten Rang zurückgesetzt, wenn spätere Jahresfelder leer sind.
- Beispiel: Tobias Huber bleibt Herzog, wenn ab 2017 Herzog eingetragen ist, auch wenn 2025 leer ist.
- Beim Import und bei manuellen Statusänderungen wird eine Rückstufung technisch verhindert.
- Die Migration schützt bestehende Lagerstatusdaten zusätzlich gegen Rückstufungen, soweit in der Datenbank bereits höhere Ränge bekannt sind.

## Hinweise

Nach dem Update sollte der Import „Zeltlager Manager 2025 importieren“ erneut ausgeführt werden. Bestehende Personen werden wiederverwendet. Höhere bekannte Ränge und Beinamen werden nachgetragen.

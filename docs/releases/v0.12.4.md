# Release v0.12.4

## Ziel

Hotfix für die Erstinstallation über `public/setup.php`.

## Ursache

Die Migration `2026_06_25_000012_create_imports` verwendete in `import_run_errors` die Spalte `row_number`. Auf dem Zielsystem führt das bei MySQL/MariaDB zu einem SQL-Syntaxfehler, weil `ROW_NUMBER` als SQL-Funktion bzw. reservierter Ausdruck behandelt werden kann.

## Umsetzung

Die Spalte wurde auf `source_row_number` umbenannt. ImportService und Ansicht wurden entsprechend angepasst.

## Geänderte Dateien

- `database/migrations/2026_06_25_000012_create_imports.php`
- `app/Services/ImportService.php`
- `app/Views/imports/show.php`
- `VERSION`
- `CHANGELOG.md`
- `docs/releases/v0.12.4.md`

## Datenbank

Keine neue Migration. Die bestehende Import-Migration wurde korrigiert, weil sie beim Setup noch nicht erfolgreich abgeschlossen wurde.

## Tests

- PHP-Syntaxprüfung für alle PHP-Dateien.
- ZIP-Prüfung.
- Suche auf verbliebene unquotierte `row_number`-SQL-Verwendung.

## Bekannte Einschränkungen

Wenn die Tabelle `import_run_errors` manuell bereits mit der Spalte `row_number` angelegt wurde, sollte sie vor erneutem Setup gelöscht oder manuell angepasst werden. Beim gemeldeten Fehler wurde die Tabelle typischerweise nicht erstellt.

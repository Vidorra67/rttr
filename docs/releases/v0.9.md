# Release v0.9.0 – Ordnungspunkte und mobiler Punkteabzug

## Ziel

Diese Version ergänzt den Ritterlager Manager um eine mobile Ordnungspunkte-Funktion. Normale Mitarbeiter können einem Kind Punkte abziehen. Die Kategorie ist serverseitig fest auf Ordnung begrenzt.

## Neu

- Neue Ansicht `/ordnung` für den mobilen Punkteabzug.
- Neue Adminansicht `/admin/ordnungspunkte` für alle Ordnungseinträge.
- Neue Korrekturansicht `/admin/ordnungspunkte/korrektur` für Lagerleitung und Admins.
- Storno statt Löschung für Punkteeinträge.
- Übersichtskarte „Ordnung heute“ auf dem Dashboard.
- Navigationseintrag „Ordnung“ für berechtigte Nutzer.

## Datenbank

Neue Migration:

```text
database/migrations/2026_06_25_000010_create_point_entries.php
```

Neue Tabellen:

- `point_categories`
- `point_entries`

Seed-Kategorien:

- Ordnung
- Spiel
- Wettbewerb
- Prüfung
- Bonus

Normale Mitarbeiter dürfen nur Ordnung über den mobilen Flow buchen.

## Sicherheit

- `points.order.create` erlaubt den mobilen Ordnungspunkte-Abzug.
- `points.manage` erlaubt Liste, Korrekturen und Storno.
- Die Kategorie Ordnung wird serverseitig erzwungen und nicht aus POST übernommen.
- Normale Mitarbeiter dürfen nur -1, -2 oder -3 buchen.
- Admins und Lagerleitung können Korrekturen buchen.
- Einträge werden nicht hart gelöscht.
- CSRF ist für Abzug, Korrektur und Storno aktiv.
- Audit-Log protokolliert Anlage und Storno.

## Update

Nach dem Einspielen ausführen:

```bash
php scripts/maintenance/migrate.php
```

Danach einmal neu anmelden, damit die Session-Rechte frisch geladen werden.

## Tests nach Update

- Login als Mitarbeiter.
- `/ordnung` öffnen.

- -1 Ordnungspunkt mit Grund buchen.
- Manipulierten POST mit anderer Kategorie prüfen. Kategorie darf nicht übernommen werden.
- Login als Admin oder Lagerleitung.
- `/admin/ordnungspunkte` öffnen.
- Eintrag stornieren.
- Korrektur buchen.
- Audit-Log prüfen.

# Release v0.2.0: Personen, Rollen und PIN-Login

## Ziel

Diese Version führt das einheitliche Loginmodell für den Ritterlager Manager ein. Alle Benutzer melden sich über eine Personenauswahl und eine 4 bis 6 stellige PIN an. Adminrechte sind Rollenrechte und kein eigener Loginweg.

## Enthalten

- Personenbasis
- Benutzerzugang je Person
- Rollen und Berechtigungen
- Login per Dropdown und PIN-Touchpad
- Logout
- Personenverwaltung
- Rollenzuweisung
- Login aktivieren und deaktivieren
- PIN setzen und ändern
- Login-Fehlversuche mit Sperrzeit
- Audit-Log für Login, Logout, Rollen und PIN

## Nicht enthalten

- keine Mandantenfähigkeit
- keine vollständige Teilnehmerdetailmaske
- kein Speiseplan
- keine Dienstplanung
- keine Ordnungspunkte
- kein Import

## Update

1. Backup der vorhandenen Version erstellen.
2. Dateien aus v0.2.0 übertragen.
3. Produktive `config/app.php` und `config/database.php` nicht ungewollt überschreiben.
4. Migrationen ausführen:

```bash
php scripts/maintenance/migrate.php
```

5. Ersten Admin anlegen, falls noch kein loginfähiger Benutzer existiert:

```bash
php scripts/maintenance/create_user.php --first=Max --last=Mustermann --pin=123456 --role=admin
```

6. Danach im Browser anmelden und die PIN sofort ändern.

## Tests nach Update

- `/login` öffnet die Loginseite.
- Login-Dropdown zeigt aktive loginfähige Personen.
- Login mit korrekter PIN funktioniert.
- Login mit falscher PIN wird neutral abgelehnt.
- Session-ID wird nach Login rotiert.
- Admin sieht den Menüpunkt Personen.
- Mitarbeiter sieht keine Admin-Funktionen.
- Direkter Zugriff ohne Recht liefert 403 oder Weiterleitung zum Login.
- PIN-Hash steht in der Datenbank, keine Klartext-PIN.
- Audit-Log enthält Loginereignisse.

## Rollback

Rollback ist nur über ein vollständiges Backup vor dem Update sauber möglich. Für diese Migration ist kein automatisches Downgrade vorgesehen.
